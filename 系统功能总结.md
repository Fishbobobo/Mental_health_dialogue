# 中小学生心理健康对话系统 — 功能总结

> **熊猫盼达**：基于 LLM 驱动的 Agentic 架构，为中小学生提供 AI 心理支持对话

---

## 一、系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                     用户端 (微信小程序)                       │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTP REST API
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                 Flask Web 服务器 (app.py)                     │
│  8个端点：/login /textchat /voiceChat /getHistoryChat /...    │
└──────────────────────────┬──────────────────────────────────┘
                           │ run_agent_turn()
                           ▼
┌─────────────────────────────────────────────────────────────┐
│              Agent 编排器 (agent.py)                          │
│  ┌──────────────────────────────────────────────────────┐   │
│  │ Agentic Loop (最多 8 次迭代)                          │   │
│  │                                                      │   │
│  │  LLM (qwen-plus)  ←→  Tool Calls  ←→  Handler       │   │
│  │       ↓ stop                                           │   │
│  │  final_reply                                           │   │
│  └──────────────────────────────────────────────────────┘   │
│  ● 注入 System Prompt (记忆状态 + 13 个工具定义)              │
│  ● 自动注入 recent_dialogue / summary 参数                    │
│  ● 每 5 轮异步触发对话摘要                                   │
└──────────────────────────┬──────────────────────────────────┘
                           │ handle_tool_call()
                           ▼
┌─────────────────────────────────────────────────────────────┐
│              工具分发层 (tools/tool_handler.py)               │
│                                                              │
│  评估工具 → symptom_judge / symptom_select / intent_recognize │
│            topic_search / topic_judge                        │
│  回复工具 → chat_reply / empathetic_reply / symptom_reply    │
│            topic_reply / end_reply                            │
│  记忆工具 → read_user_memory / write_user_memory             │
│            update_conversation_summary                       │
└─────────────────────────────────────────────────────────────┘
```

### 数据流

```
用户消息 → app.py 记录到 MySQL → run_agent_turn()
  → load_user_memory() 加载用户状态
  → get_history_chat() 获取对话历史
  → Agentic Loop: LLM ↔ 工具调用
  → save_user_memory() 保存用户状态
  → 异步摘要更新 (每5轮)
→ app.py 记录 AI 回复到 MySQL → 返回给前端
```

## 二、数据库结构 (7 张表)

### `student_info` — 用户基本信息
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(255) PK | 用户标识（手机号） |
| password | VARCHAR(255) | 登录密码 |
| api_key | VARCHAR(255) | 分配的 LLM API 密钥 |
| last_login_time | DATETIME | 最后登录时间 |
| topic | INT | 当前话题编号 |
| cur_selected_slot | INT | 当前症状问题编号 |
| status | INT | 对话阶段：1=评估, 2=自由聊天, 3=话题讨论 |

### `instant_info` — 会话状态
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(255) PK | 用户标识 |
| dialogue | MEDIUMTEXT | JSON 格式的对话历史 |
| turn | INT | 当日剩余轮数 (最大值 30) |
| filter_dialogue | MEDIUMTEXT | 过滤后对话（新架构未使用） |
| summary | TEXT | 对话摘要文本 |

### `PHQ-9` — 抑郁量表（9 题）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(255) PK | 用户标识 |
| q1..q9 | INT | 0-3 分，-1 表示未评估 |

### `GAD-7` — 焦虑量表（4 题，原始 7 题合并为 4 题）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(255) PK | 用户标识 |
| q1..q4 | INT | 0-3 分，-1 表示未评估 |

### `topic` — 六大生活话题
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(255) PK | 用户标识 |
| t1..t6 | INT | -1=未评估, 0=无困扰, 1=存在困扰 |

话题映射：
- t1: 家庭因素
- t2: 学业压力
- t3: 社交压力
- t4: 学校环境
- t5: 网络暴力
- t6: 自身因素

### `result` — 诊断结果（看板用）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(255) PK | 用户标识 |
| depression | VARCHAR(255) | 无/轻度/中度/中重度/重度抑郁 |
| anxiety | VARCHAR(255) | 无/轻度/中度/重度焦虑 |
| extracted_info | VARCHAR(255) | 存在困扰的话题名称 |
| serious_attention | VARCHAR(255) | 是否需要重点关注 |

### `code_num` — 短信验证码
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(255) PK | 手机号 |
| code | VARCHAR(255) | 6 位验证码 |
| timestamp | BIGINT | 过期时间戳（5 分钟有效） |

## 四、Agentic Loop 工作流程

### `run_agent_turn(cursor, user_name) → str`

```
1. 加载用户记忆
   - load_user_memory() 从 5 张 MySQL 表聚合用户状态
   - 返回包含 phq9/gad7/topic 分数、阶段、轮数等的 dict

2. 获取对话历史
   - get_history_chat() 获取对话记录
   - 截取最近 20 轮
   - 转换为 OpenAI messages 格式

3. 注入 System Prompt
   - memory_to_context() 将记忆序列化为文本
   - 格式化 SYSTEM_PROMPT_TEMPLATE
   - 注入 13 个工具定义

4. Agentic Loop（最多 8 轮）
   ┌─────────────────────────────────────┐
   │ LLM (qwen-plus, temperature=0.7)    │
   │  → finish_reason="stop": 输出最终回复│
   │  → finish_reason="tool_calls":      │
   │     自动注入参数 → handle_tool_call │
   │     追加 tool result → 继续循环     │
   └─────────────────────────────────────┘

5. 保存更新后的用户记忆到 MySQL

6. 每 5 轮异步触发对话摘要更新
```

### 自动参数注入

当 LLM 调用工具时，`agent.py` 自动注入：
- `recent_dialogue`：JSON 格式的最近对话（filter_info 格式）
- `summary`/`summary_before`：最新的对话摘要文本

基于各工具的 `parameters.properties` 自动判断。

## 五、13 个 LLM 工具定义

### 评估工具 (5 个)

| 工具名称 | 功能 | 调用方式 |
|----------|------|----------|
| `assess_symptom` | 评估 PHQ-9(1-9)/GAD-7(10-13) 症状严重程度 | 返回 -1~3 分 |
| `select_next_symptom` | 选择下一个最合适的症状问题 | 返回问题编号 |
| `recognize_intent` | 判断用户意图（想聊天/接受评估） | 返回 0/1/3 |
| `search_topic` | 判断当前对话是否匹配预定义话题 | 返回话题编号 + 是否匹配 |
| `judge_topic` | 评估用户在某个话题上的困扰程度 | 返回 -1/0/1 |

### 回复工具 (5 个)

| 工具名称 | 功能 | 使用场景 |
|----------|------|----------|
| `generate_chat_reply` | **主要回复工具**：生成符合咨询阶段的回复 | 绝大多数场景：开场共情、深入探索、自由聊天、百科问答等 |
| `generate_symptom_reply` | 询问某个具体的 PHQ-9/GAD-7 症状问题 | 评估阶段使用 |
| `generate_empathetic_reply` | 深度共情：引导探索情绪（不给建议） | 用户表达中重度情绪困扰时 |
| `generate_topic_reply` | 围绕六大主题讨论 | 话题讨论阶段 |
| `generate_end_reply` | 对话总结结束语 | 用户找到解决方案或情绪好转后 |

### 记忆工具 (3 个)

| 工具名称 | 功能 |
|----------|------|
| `read_user_memory` | 查看用户完整状态信息 |
| `write_user_memory` | 更新用户记忆字段（阶段、当前问题等） |
| `update_conversation_summary` | 更新对话摘要 |

## 六、System Prompt 核心原则

**角色**：你是盼达，一只温暖的心理陪伴熊猫

### 四大核心原则

```
原则一：少说多听
  - 每轮回复 ≤ 2 句话，总字数 ≤ 80 字
  - 多使用简短回应："嗯""然后呢""能再多说说吗"
  - 宁短勿长

原则二：不是每次都要提问
  - 可用纯陈述句：共情、总结、点头式回应
  - 如果提问：只能问 1 个开放式问题
  - 封闭式问题（"是不是""有没有"）禁止使用

原则三：不直接给建议
  - 禁止说"你应该…""我建议你…""你可以试试…"
  - 通过开放式提问引导用户自己找到答案
  - 用户感受应是"我自己想通的"，不是"盼达告诉我的"

原则四：避免重复句式
  - 每轮必须更换开头方式
  - ✅ 允许的开头：轻声回应、重复用户关键词、
    直接说感受、简短肯定、比喻式表达、直接总结
  - ❌ 禁止："这件事" 开头（模板化严重）
```

### 对话四阶段

```
阶段一：开场共情
  目标：让用户感受到被接纳，愿意开口
  方法：引导描述具体事件和内心感受，不急于评估

阶段二：深入探索
  目标：理清"发生了什么→什么感受→为什么"
  方法：用户抵触时不说"不想说就不说"，而是温和陪伴

阶段三：认知重构
  目标：提供新视角，帮助用户重新看待问题
  方法：温和面质"你刚才说…但我观察到…你觉得有关系吗？"

阶段四：结束总结
  目标：总结用户自己发现的解决方案，强化成就感
  注意：不是每轮都需要结束，只在合适时机
```

## 七、13 个 Tool Handler 详解

### `_assess_symptom`
- 调用 `symptom_judge(q_index, dialogue_list, api_key)`
- PHQ-9(1-9) 更新 `phq9_scores`；GAD-7(10-13) 更新 `gad7_scores`
- Q9 ≥ 1 时自动设置 `suicide_risk = True`

### `_select_next_symptom`
- `phq_gad_unfill_slot()` 获取未填写的症状编号
- `symptom_select()` 选择最相关的下一个问题
- 更新 `current_question`

### `_recognize_intent`
- `intent_judge()` 判断意图
- 所有症状评估完且用户无话题 → 返回 3（对话已全部完成）

### `_search_topic`
- `topic_search()` 匹配六大话题
- 匹配成功时更新 `current_topic`

### `_judge_topic`
- `topic_judge()` 评估困扰程度
- 更新 `topic_scores[t_index]`

### `_generate_chat_reply`
- 调用 `chat_reply(dialogue_list, summary, api_key)`
- 设置 phase="free_chat"
- **最常用的回复工具**

### `_generate_empathetic_reply`
- 调用 `empathetic_reply(question_index, dialogue_list, summary, api_key)`
- 设置 phase="free_chat"

### `_generate_symptom_reply`
- 调用 `symptom_ask_reply(question_index, dialogue_list, summary, api_key)`
- 设置 phase="assessment"

### `_generate_topic_reply`
- 调用 `topic_reply(topic_index, dialogue_list, summary, api_key)`
- 设置 phase="topic_discussion"

### `_generate_end_reply`
- 调用 `end_reply(dialogue_list, api_key)`
- 设置 phase="ending"

### `_read_user_memory`
- 返回 `memory_to_context(memory)` 文本

### `_write_user_memory`
- 更新 memory dict 中指定的字段

### `_update_conversation_summary`
- 调用 `generate_summary()` 更新 `memory["summary"]`

## 八、回复模块详情

### `chat_reply.py` — 自由聊天回复
- **输入**：`chat_reply(dialogue_history, summary, key)`
- **输出**：温暖、简短（≤50字）、1-2句话
- **特点**：
  - 每轮追踪上轮 assistant 回复开头，强制变换句式
  - 提供 6 种好的回复示例
  - 绝对禁止"这件事"、"听起来…这件事…"组合
  - 通过引用用户关键词自然开场

### `empathetic_reply.py` — 共情回复
- **输入**：`empathetic_reply(index, dialogue_history, summary, key)`
- **输出**：包含策略标注的共情回复（≤60字）
- **策略池**（随机排序，LLM 自主选择）：
  - **探索(EXPLORE)**：通过提问引导深入探索
  - **建立联系(CONNECTION)**：赞同、安慰、鼓励
  - **反馈(FEEDBACK)**：赞赏、反馈感受
- 关注学生的兴趣爱好和生活闪光点

### `symptom_reply.py` — 症状询问回复
- **输入**：`symptom_ask_reply(index, dialogue_history, summary, key)`
- **策略组合**（各选一个组合）：
  - 话题过渡策略：评论+转移(Comment then Shift)、桥接(Bridging)
  - 提问策略：加载式问题、提名式技巧、宽恕式问题、澄清式问题
- 自然地从当前话题过渡到症状问题

### `topic_reply.py` — 话题讨论回复
- **输入**：`topic_reply(index, dialogue_history, summary, key)`
- **输出**：围绕六大话题的讨论回复（≤80字）

### `end_reply.py` — 结束回复
- **输入**：`end_reply(dialogue_history, key)`
- **输出**：结尾总结，不提新问题或新话题

## 九、评估模块详情

### `symptom_judge.py` — 症状评估
- **流程**：LLM 判断 → 规则匹配 → 默认返回 0（无症状）
- **规则**：基于频率关键词（从未/偶尔/经常/每天）映射严重程度
- **分数**：-1(无法判断) / 0(无症状) / 1(轻度) / 2(中度) / 3(重度)

### `symptom_select.py` — 问题选择
- **流程**：LLM 根据当前对话选择最切合的症状问题
- **回退**：随机选一个未评估问题

### `intent_recognize.py` — 意图识别
- **输出**：0=接受评估，1=有想聊的话题
- **判断依据**：用户最后一句是否包含新话题、新问题或情绪表达

### `topic_search.py` — 话题匹配
- 6 个预定义话题，匹配最近 4 轮对话
- LLM 输出话题名称，映射到编号

### `topic_judge.py` — 话题困扰判断
- -1(无法判断) / 0(无困扰) / 1(存在困扰)

## 十、记忆管理 (memory.py)

### 用户记忆模型

```python
memory = {
    "user_id": str,           # 用户标识
    "turn_count": int,        # 已对话轮数（从 MAXTURN - DB_turn 计算）
    "phq9_scores": {          # PHQ-9 每题得分，共 9 题
        "Q1": -1, ... "Q9": -1
    },
    "gad7_scores": {          # GAD-7 每题得分，共 4 题
        "Q10": -1, ... "Q13": -1
    },
    "topic_scores": {         # 话题困扰评估，共 6 个
        "t1": -1, ... "t6": -1
    },
    "phase": "assessment",    # 当前对话阶段
    "current_question": None, # 当前症状问题编号
    "current_topic": None,    # 当前话题编号
    "summary": "",            # 对话摘要
    "suicide_risk": False,    # 自杀风险标记
}
```

### 阶段映射

| DB status | phase | 说明 |
|-----------|-------|------|
| 1 | assessment | 症状评估阶段 |
| 2 | free_chat | 自由对话阶段 |
| 3 | topic_discussion | 话题讨论阶段 |
| 4 | ending | 结束阶段（由 generate_end_reply 设置，不可逆） |

### 序列化示例 (`memory_to_context`)

```

PHQ-9 评估进度: Q1=2(中), Q3=1(轻), Q9=0(无)

```

## 十一、对话流程优化要点

### 多轮回复质量检查（E2E 测试标准）

| 检查项 | 阈值 |
|--------|------|
| 回复长度 | 4-80 字 |
| 违禁句式 | 不含"这件事" |
| 问题数量 | 最多 1 个问号（？） |
| 封闭式问题 | 不含"是不是/有没有/对不对/能不能/会不会" |
| 直接建议 | 不含"你应该/我建议你/你可以试试/你最好" |

### 句式多样性要求

每轮回复必须使用不同的开头方式，避免连续多轮同句式：

- ✅ 轻声回应：`"嗯…""这样啊""我明白了"`
- ✅ 引用关键词：`"考不好……""怕他们失望……"`
- ✅ 直接说感受：`"我感受到你……""我能理解……"`
- ✅ 简短肯定：`"你说得对""确实不容易"`
- ✅ 接用户的话：直接回应用户最后一句的关键词
- ❌ 禁止："这件事…"（任何情况下）
- ❌ 避免："听起来…"（除非唯一表达方式）
